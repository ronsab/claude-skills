# which-skill — איזה סקיל מתאים לצורך שלי?

## מטרה
רון מתאר צורך בשפה חופשית, וקלוד מנתח לעומק ומחזיר: האם קיים סקיל מתאים, ואם לא — מה ההמלצה (כולל הצעה ליצור סקיל חדש).

זו הגרסה החכמה של "מתאם הצרכים" שבמפה (`skill-map.html`). המפה נותנת מענה מיידי מבוסס מילות מפתח; הפקודה הזו נותנת ניתוח עם הבנה אמיתית.

הצורך מגיע כ-argument: **$ARGUMENTS**

---

## ביצוע

### שלב 1: טען את מסד הידע
קרא את `C:\Users\USER\.claude\skill-map\skill-map-data.json`. הוא מכיל:
- `skills` / `commands` / `agents` — הסקילים העסקיים עם whenToUse, output, opportunity
- `decisionRules` — מיפוי intent → skill + מה לומר
- `recipes` — שרשראות סקילים (מתכונים) לתוצרים גדולים
- `conflicts` — סקילים חופפים + כלל הניתוב
- `enrich` — דוגמאות וטוויסטים לכל סקיל

אם הצורך לא ברור מהסקילים העסקיים, אפשר גם לסרוק `~/.claude/skills/` לסקילים שלא ב-curation.

### שלב 2: נתח את הצורך
בהתבסס על התיאור של רון, קבע:

1. **התאמה ישירה** — יש סקיל שעונה בדיוק על הצורך?
   → ציין את שמו + `invoke` + דוגמה (מ-`enrich`) + מתי להשתמש.

2. **כמה מועמדים** — כמה סקילים יכולים להתאים?
   → המלץ על הטוב ביותר לפי `decisionRules`/`conflicts`, וציין חלופות בקצרה.

3. **מתכון מתאים יותר** — הצורך דורש כמה סקילים יחד?
   → הצע את ה-`recipe` הרלוונטי (השרשרת המלאה).

4. **אין התאמה** — אין סקיל קיים?
   → ציין את הקרוב ביותר, והמלץ ליצור סקיל חדש עם `skill-creator`. הצע **שם** ו-**description** ראשוני לסקיל החדש.

### שלב 3: תשובה
בעברית, קצרה וממוקדת. מבנה:
```
🎯 הצורך: [סיכום במשפט]

✅ הסקיל המתאים: [שם]
   להפעלה: [מה לומר / הפקודה]
   מתי: [הקשר קצר]

[אם רלוונטי] חלופות: [סקיל אחר] — [מתי עדיף]
[אם מתכון] מתכון מומלץ: [שרשרת]
[אם אין] אין סקיל קיים. הקרוב: [X]. המלצה: ליצור סקיל '[שם מוצע]' עם skill-creator.
```

## כללי זהב
- אל תמליץ על סקיל שלא קיים במסד או בתיקיית הסקילים — בדוק קודם.
- אם יש סתירה ידועה (`conflicts`), השתמש בכלל הניתוב שכבר הוגדר.
- העדף סקילים מקומיים (local/CANONICAL) על-פני כפילויות מ-plugins.
- אם הצורך עסקי-שיווקי, שקול גם הזדמנות מ-`opportunities` ומתכון מ-`recipes`.
- אם רון מתאר משהו חוזר שאין לו סקיל — זו הזדמנות: הצע ליצור סקיל-לקוח/שירות.
