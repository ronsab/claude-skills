# Deployment & Embedding — מדריך דיפלוי והטמעה

## Vercel — אחסון סטטי

### מתי להשתמש
תמיד. כל דף נחיתה שנבנה חייב לעלות ל-Vercel לפני הטמעה ב-LANDY.

### מבנה תיקייה נדרש
```
project-name/
  index.html      ← הדף הראשי
  hero-bg.jpg     ← תמונות (אם יש)
  package.json     ← מינימלי, אם לא קיים
```

### package.json מינימלי
```json
{
  "name": "project-name",
  "version": "1.0.0"
}
```

### דיפלוי
```bash
# אופציה 1: MCP tool
deploy_to_vercel(projectPath, projectName)

# אופציה 2: Vercel CLI
vercel deploy --prod
```

### בדיקה לאחר דיפלוי
1. פתח את ה-URL שהתקבל בדפדפן/Playwright
2. ודא שכל הסקשנים נטענים
3. ודא שתמונות מוצגות
4. ודא שאין שגיאות ב-console

## LANDY — מערכת דפי נחיתה

### מגבלות קריטיות
| מגבלה | פירוט |
|--------|--------|
| סוג מערכת | AI-based — מפרש טקסט כהוראות, לא כקוד |
| שדה תיאור | עד 800 תווים |
| שדה הודעה | עד 5,000 תווים |
| קוד HTML | **לא מקבל** — אם מדביקים קוד, LANDY מגיב עליו כאילו זו שיחה |
| קבצים חיצוניים | לא ניתן להעלות קבצי HTML |

### הפתרון: iframe דרך Vercel

במקום להדביק קוד, שולחים ל-LANDY הוראה בשפה טבעית שמבקשת ממנו להטמיע iframe.

### הודעה מוכנה להדבקה ב-LANDY

**החלף `[VERCEL_URL]` בכתובת האמיתית:**

```
מחק את כל התוכן הקיים בדף. הטמע במקומו iframe שמציג את הכתובת [VERCEL_URL] בלבד. ה-iframe צריך לתפוס 100% רוחב ו-100% גובה של המסך, ללא גבולות, ללא שוליים, וללא header או footer של LANDY. הקוד: <iframe src="[VERCEL_URL]" style="width:100%;height:100vh;border:none;margin:0;padding:0" allowfullscreen></iframe>
```

### יתרון שיטת iframe
- כל עדכון שעולה ל-Vercel מתעדכן אוטומטית ב-LANDY
- לא צריך לגעת ב-LANDY שוב אחרי ההטמעה הראשונית
- אין מגבלת גודל — הדף כולו נטען מ-Vercel

## Lovable — לפרויקטים מורכבים

### מתי להשתמש
כשהלקוח צריך יותר מדף נחיתה סטטי:
- אפליקציה עם התחברות משתמשים
- דשבורד
- מערכת הזמנות
- פרויקט עם מסד נתונים

### קשר לסקיל
הסקיל הזה (landing-page-builder) מתמקד בדפי נחיתה סטטיים בלבד.
לפרויקטים מורכבים — הפנה את המשתמש ל-Lovable.
