# Base44 Business System Builder — מערכות ניהול עסקיות לעסקים ישראליים

## עקרונות הסקיל

אתה בונה **מערכת ניהול מלאה ומבוססת-נתונים** על **Base44** (פלטפורמה מנוהלת: entities, auth,
roles, hosting, server functions). התוצר אינו קובץ סטטי אלא **אפליקציה חיה** עם בסיס נתונים,
משתמשים והרשאות, שעובדת ב-Web ובנייד (PWA).

**מתאים ל:** מערכות הזמנות, מלאי/מחסן (WMS), CRM, ERP קל, פורטל לקוחות, ניהול סוכני שטח.
**לא מתאים ל:** דף נחיתה שיווקי (השתמש ב-`landing-page-builder`) או מצגת (`ron-digital-presentation`).

**אילו כלי MCP בשימוש:** `create_base44_app`, `create_entity_schema`, `create_entities`,
`update_entities`, `query_entities`, `list_entity_schemas`, `edit_base44_app`.

**ארבעה עקרונות-על** (קרא `references/build-workflow.md` לפרטים):
1. **פלטפורמה מנוהלת** — אפס DevOps, תחזוקה מינימלית לטווח ארוך.
2. **נייד = PWA** — אותה אפליקציה, ללא pipeline נייטיב נפרד.
3. **לוגיקה רגישה ב-server functions בלבד** — תמחור, שריון מלאי, מעברי-סטטוס, מספור מסמכים. עוקף את היעדר הטרנזקציות ב-Base44.
4. **Snapshot של מחיר/מע"מ** בשורות מסמך — מסמכים היסטוריים נשארים immutable.

## שלב 1 — שאלון גילוי

אסוף לפני בנייה (אם כבר נאמר בשיחה — חלץ, אל תשאל שוב). מטרה: סבב שאלות אחד.

- **משתמשים/תפקידים**: מי משתמש במערכת? (בעלים, תפעול, סוכן שטח, מחסנאי, מלגזן, נהג, הנה"ח, לקוח עסקי/פרטי...)
- **מודל תמחור**: מחירים לפי סוג לקוח? לפי סוכן? מחיר ספציפי ללקוח? הנחת-סוכן עם תקרה?
- **מסמכים נדרשים**: הצעות מחיר? תעודות משלוח? חשבוניות/קבלות? (← קובע אינטגרציית ספק)
- **גודל קטלוג**: כמה פריטים? (מאות → אלפים משפיע על חיפוש/עימוד)
- **פלטפורמה**: Web בלבד או Web+נייד?
- **תהליך**: ההזמנה נשלחת אוטומטית למחסן? מלאי בזמן אמת?

## שלב 2 — המלצת פלטפורמה ועקרונות

## שלב 4 — מודל הנתונים

קרא `references/data-model.md`. Base44 בונה ישויות בסיס אוטומטית; אמת ב-`list_entity_schemas`
והשלם/תקן דרך `create_entity_schema`. ישויות ליבה לעסק ישראלי: Product, Category, Inventory,
StockMovement, PriceList/Item, Customer (עסקי/פרטי), Agent, Order/OrderLine, Quote, DeliveryNote,
DocumentRef, Payment, Counter, User+roles. הקשר: ש"ח, מע"מ 18%, ח.פ, תנאי תשלום.

## שלב 6 — לוגיקת שרת (server functions)

קרא `references/build-workflow.md` (סעיף server functions). המרכזיות:
- `resolvePrice` — תמחור דטרמיניסטי לפי קדימות מחירונים + הנחת-סוכן עם clamp לתקרה + מע"מ, ושמירת snapshot.
- `reserveInventory` — **מלאי בזמן אמת + מניעת מכירת-יתר** (optimistic concurrency + ledger).
- מספור רציף דרך `Counter` (order_number / quote_number / delivery_note_number).
- `submitOrder` — ניתוב לאישור בחריגת הנחה + **שליחה אוטומטית למחסן**.

## שלב 7 — מודול מסמכים

קרא `references/israeli-compliance.md`. **גישה משולבת**:
- **בתוך המערכת** (PDF RTL): הצעת מחיר (אישור → `convertQuoteToOrder`), תעודת משלוח.
- **דרך ספק מורשה (Green Invoice)**: חשבונית מס, קבלה, זיכוי → נשמרים כ-`DocumentRef` עם מספר הקצאה.
- **אסור** להנפיק חשבונית/קבלה חוקית בתוך המערכת.

## שלב 8 — זריעת נתוני דמו

דרך `create_entities`: 10–20 מוצרים **עם תמונות** (גלריה ריקה כש-`image_url` ריק!), לקוח עסקי
+ פרטי, מחירונים, והזמנה/הצעה/תעודת משלוח/תשלום לדוגמה — כדי שהלקוח יראה מערכת מלאה מיד.

---

## שלב 9 — עריכה אינקרמנטלית ואימות

עבוד **צעד-אחר-צעד, בבטחה**. `edit_base44_app` עם מילות המשתמש; **לא לשלוח עריכה חדשה לפני
שהקודמת נקלטה ואומתה**. אמת כל שלב דרך `list_entity_schemas` / `query_entities`. החזר editorUrl
בכל edit. תעד שער-אימות לכל שלב במפת הדרכים.

## טעויות נפוצות למנוע

| בעיה | פתרון |
|---|---|
| גלריה מציגה placeholders אפורים | ודא `image_url` מאוכלס לכל מוצר; הוסף מנגנון העלאת תמונה במסך ניהול המוצרים |
| שדות חובה ב-User חוסמים תפקידים | אל תגדיר `assigned_warehouse`/`vehicle_number` כ-required לכל המשתמשים — רק רלוונטי לנהג/מחסנאי |
| עריכות חופפות שוברות מה שעבד | המתן שכל `edit_base44_app` ייקלט ויאומת לפני העריכה הבאה |
| מכירת-יתר (שני סוכנים, יחידה אחרונה) | כל שינוי מלאי דרך server function יחיד + ledger; אל תעדכן מלאי מצד-לקוח |
| מחיר מחושב רק בצד-לקוח | `resolvePrice` בצד-שרת + snapshot ב-OrderLine |
| הנפקת חשבונית חוקית במערכת | אסור — דרך ספק מורשה (Green Invoice); שמור `DocumentRef` בלבד |
| מספור מסמכים עם דילוגים | רצף דרך `Counter` ב-server function, לא ספירה מקומית |
| מילות appPrompt מורחבות | העבר את מילות המשתמש כפי שהן — ל-Base44 AI משלו להרחבה |
