# תאימות ישראלית — מע"מ, מטבע ומסמכים

## מטבע ומע"מ
- מטבע: **ש"ח (ILS)**.
- מע"מ: **18%** (ברירת מחדל; שמור כשדה ניתן-לעדכון פר-מוצר/מערכת, לא קבוע בקוד — השיעור משתנה לאורך זמן).
- מוצרים פטורים: אפשר `is_vat_exempt`/`vat_included` פר-מוצר (אילת, מקרים מסוימים).
- חישוב: `line_ex_vat = unit_price × qty` ; `vat = line_ex_vat × vat_rate` ; `total = ex_vat + vat`.

## מסמכים — גישה משולבת (חובה להבין את החלוקה)

| מסמך | היכן מופק | חוקי? | הערות |
|---|---|---|---|
| הצעת מחיר | **בתוך המערכת** (PDF RTL) | לא מחייב | אישור → המרה אוטומטית להזמנה |
| תעודת משלוח | **בתוך המערכת** (PDF RTL) | מספור פנימי רציף | snapshot של כמות שנמסרה |
| חשבונית מס | **ספק חיצוני (Green Invoice)** | חוקי — מספר הקצאה | נשמר כ-DocumentRef |
| קבלה / חשבונית מס-קבלה | **ספק חיצוני** | חוקי | רישום Payment → הנפקה דרך הספק |
| זיכוי | **ספק חיצוני** | חוקי | בהחזרות |

**כלל ברזל:** אין להנפיק חשבונית מס/קבלה חוקית בתוך המערכת. בישראל נדרשים מספר הקצאה
(חשבונית ישראל / clearance בזמן אמת מול רשות המסים) וספק מורשה. המערכת היא **התפעולית**;
הספק הוא **מערכת הרשומה החוקית**.

## אינטגרציית Green Invoice (חשבונית ירוקה)
- ספק ברירת-המחדל המומלץ. דורש **API key** (הוסף מסך הגדרות להזנתו — אל תמציא מפתח).
- זרימה: הזמנה מאושרת/נמסרה → server function `issueInvoice` → קריאת API → קבלת
  `external_id`, `allocation_number`, `pdf_url` → שמירת `DocumentRef`.
- קבלה: רישום `Payment` → `issueReceipt` → DocumentRef(doc_type=קבלה/חשבונית מס-קבלה).
- ספקים חלופיים מקובלים: חשבשבת, ריווחית, Priority, EZcount — אותו דפוס DocumentRef, רק ה-API משתנה.

## מספור רציף (דרישה חוקית)
מספרי מסמך פנימיים (הצעת מחיר, תעודת משלוח) חייבים רצף ללא דילוגים — דרך `Counter` ב-server
function מוגן. מספרי חשבונית/קבלה חוקיים מנוהלים ע"י הספק.
