# video-merge - איחוד קליפים לסרטון אחד

מאחד קבוצת קליפים (כל אחד עד ~10 שניות) לסרטון אחד של עד 90 שניות, מוכן לאינסטגרם/פייסבוק.

## דרישות סביבה

- `ffmpeg` - הסקריפטים מאתרים אותו לבד (PATH או תיקיית WinGet)
- Python - תמיד בנתיב המלא (alias של python שבור ב-Windows של רון):

```
C:\Users\USER\AppData\Local\Programs\Python\Python311\python.exe
```

- הרץ מתוך תיקיית `scripts` של הסקיל (בגלל import של `_util`)

## זרימת עבודה

### 1. רשימת קבצים וסדר
קבל מרון את הקבצים בסדר הרצוי. אם נתן תיקייה - הצג לו את הקבצים שמצאת ושאל על הסדר. אל תנחש סדר.

### 2. איחוד

```
python.exe merge.py "clip1.mp4" "clip2.mp4" "clip3.mp4" --out "merged.mp4"
```

ברירות מחדל: 1080x1920 אנכי, 30fps, מקסימום 90 שניות. מה שהסקריפט עושה לבד:
- קליפ בפורמט/רזולוציה אחרת מותאם אוטומטית (הקטנה + פסים שחורים, בלי לחתוך תוכן)
- קליפ בלי אודיו מקבל פס שקט
- חיתוך ישיר בין קליפים, בלי קידוד כפול של התוצאה

### 3. חריגה מ-90 שניות
אם הסך עובר את המגבלה הסקריפט עוצר (exit code 2) ומדפיס את משך כל קליפ. הצג לרון את הפירוט ובקש החלטה: להסיר קליפ או לחרוג מהמגבלה (אפשר עם `--max-seconds` גבוה יותר רק אם רון מאשר במפורש).

### 4. אימות (חובה)
- בדוק שהמשך הסופי שהסקריפט מדפיס שווה בערך לסכום המשכים
- חלץ פריים אחד מאמצע כל מקטע והצג לרון - לוודא סדר נכון ושקליפים אופקיים ממורכזים:

```
ffmpeg -y -ss <אמצע המקטע בשניות> -i merged.mp4 -frames:v 1 check.png
```

### 5. כתוביות על התוצאה
שאל את רון: "להוסיף כתוביות לסרטון המאוחד?" אם כן - הפעל את הסקיל `hebrew-subtitles` על קובץ הפלט (הזרימה הרגילה: תמלול ← אישור ← צריבה). זה הסדר הנכון: קודם איחוד, אחר כך כתוביות, כדי שהתמלול ירוץ פעם אחת על הכל.

## פתרון תקלות

- `אודיו לא מסונכרן אחרי איחוד` - ודא שכל הקליפים עברו דרך הנרמול (הסקריפט עושה זאת תמיד; אל תאחד קבצים ידנית עם concat בלי נרמול)
- `קליפ נראה מוקטן עם פסים גדולים` - זה קליפ אופקי בתוך פריים אנכי, התנהגות מכוונת. אם רון מעדיף חיתוך למילוי מלא - שנה ב-merge.py את force_original_aspect_ratio ל-increase והוסף crop (רק באישור רון)
- `הסרטון נחתך באמצע קליפ` - בדוק אם רון העביר --max-seconds נמוך; הסקריפט לא חותך לבד, הוא עוצר עם שגיאה
