# File Dashboard

בניית דשבורד HTML מכל קובץ, לפי מה שהמשתמש באמת רוצה לראות.

ההבדל המרכזי מסקיל-דשבורד-קשיח: כאן **הבקשה של המשתמש מובילה**, לא תבנית. אותו קובץ אקסל יכול להניב דשבורד מכירות, דשבורד תקלות, או דשבורד פילוח לקוחות — תלוי מה ביקשו. תפקידך לקרוא את הבקשה, לחלץ מהקובץ רק את מה שרלוונטי לה, ולבנות דשבורד שעונה עליה.

## עקרונות שמנחים הכל

- **בקשה לפני תבנית.** קודם תבין מה המשתמש שואל. הדשבורד הוא תשובה ויזואלית לשאלה שלו, לא דאמפ של כל מה שיש בקובץ.
- **רק נתונים אמיתיים.** כל מספר, אחוז וציטוט מגיע מהקובץ. לעולם אל תמציא נתונים כדי "למלא" גרף.
- **חלץ לפני שאתה מעצב.** תמיד הרץ חילוץ אמיתי וקרא מה יש בקובץ לפני כתיבת HTML. אל תנחש מבנה.
- **קובץ אחד standalone.** פלט HTML יחיד, ללא תלות חיצונית מלבד Google Fonts. נפתח בכל דפדפן בלחיצה.

## תהליך עבודה

### שלב 1 — הבן את הבקשה, ובמידת הצורך חדד מול המשתמש
לפני שבונים, ודא שאתה יודע שני דברים. אם אחד מהם לא ברור — **שאל את המשתמש בשאלה ממוקדת אחת** (רצוי עם אופציות לבחירה), לא חקירה ארוכה. אם המשתמש כבר ענה עליהם בבקשה, או אומר "פשוט תעשה" — אל תשאל, בחר ברירת מחדל חכמה וציין בקצרה מה בחרת.

**א. מיקוד הדשבורד — מה המשתמש רוצה לראות.**
לאיזו שאלה הדשבורד עונה? מי הקהל? מה הכי חשוב?
- אם הבקשה ספציפית ("דשבורד מכירות לפי אזור", "מה הכאבים שעלו בפגישה") — תבנה לפי זה.
- אם הבקשה כללית ("תעשה דשבורד מהקובץ") — **אחרי שחילצת והצצת בנתונים** (שלב 2), הצע 2–4 זוויות קונקרטיות שבאמת קיימות בקובץ ("אפשר למקד ברווחיות לאורך השנה, בפילוח מקורות ההכנסה, או בניתוח ההוצאות — מה הכי מעניין?"), ותן גם אפשרות "תבחר אתה". כך המשתמש מדייק בלי להתאמץ, וההצעות אמיתיות ולא גנריות.

**ב. צבעים / נושא עיצובי.**
הצע למשתמש לבחור פלטה. ב-`references/design-system.md` תחת **"פלטות מוכנות"** יש כמה ערכות מוכנות (כהה-כחול ברירת מחדל, בהיר-רשמי, סגול, ירוק, חם). תן לו לבחור אחת, **או** להכניס צבעים משלו — קודי hex, צבעי מותג, או תיאור ("משהו רגוע", "בצבעי הלוגו שלי"). אם לא בוחר — ברירת המחדל היא ה-DNA הכהה (כחול-ציאן). שווה לשאול את שתי השאלות (מיקוד + צבעים) יחד בהודעה אחת.

### שלב 2 — חלץ את התוכן
הרץ את הסקריפט שמזהה סוג קובץ ומחלץ תוכן מובנה:
```bash
python "<skill_dir>/scripts/extract.py" "<נתיב_הקובץ>"
```
**Windows (סביבת רון):** ה-alias `python` שבור. השתמש בנתיב המלא:
```bash
/c/Users/USER/AppData/Local/Programs/Python/Python311/python.exe "<skill_dir>/scripts/extract.py" "<נתיב_הקובץ>"
```
מטפל ב-xlsx/xls/xlsm/csv/tsv (טבלאות: שיטים, עמודות, value_counts, סכומים/ממוצעים, דגימות), pdf/docx (טקסט + כותרות + טבלאות מוטמעות), ו-txt/md/json. הפלט: תקציר קריא + בלוק JSON בין `===JSON_START===` ל-`===JSON_END===`. עובד גם בלי pandas (נופל ל-openpyxl ל-Excel ול-csv הסטנדרטי). לקבצים גדולים אפשר `--rows N` כדי לקבל יותר שורות.

אחרי החילוץ: אם טבלאי — נתח לעומק את העמודות הרלוונטיות למיקוד (חיתוכים, סכומים, מגמות). אם מסמך — חלץ נושאים חוזרים, מספרים מהטקסט, ציטוטים מייצגים (30–200 תווים), ומסקנות.

### שלב 3 — תכנן את הסיפור
החלט בקצרה: 3–4 KPI מרכזיים · 2–5 ויזואליזציות (איזה גרף לאיזה נתון — ראה `design-system.md`) · חלוקה לטאבים לפי המיקוד (2–3 ממוקדים עדיפים על מבנה מנופח) · 3–6 תובנות שכל אחת נתמכת במספר.

### שלב 4 — בנה את ה-HTML
התחל מ-`assets/template.html` (scaffold עם מערכת העיצוב, טאבים, ורכיבים) ומלא אותו. כל ה-CSS, מתכוני הגרפים (כולל חישוב donut), ורכיבי ה-UI ב-`references/design-system.md` — **קרא אותו לפני שאתה בונה**.

**החל את הפלטה שנבחרה:** כל הצבעים הם משתני `:root` ב-CSS, אז החלפת ערכת צבעים = החלפת בלוק ה-`:root` בלבד (העתק מהפלטה המתאימה ב-design-system). אם המשתמש נתן צבעי מותג — בנה `:root` משלך: הצבע הראשי → `--accent1`/הדגשות, רקע כהה או בהיר בהתאם, והשלם accents משלימים. **תמיד שמור ניגודיות מספקת לקריאוּת** (טקסט בהיר על רקע כהה או להפך) — פלטה יפה שלא קריאה היא כישלון.

### שלב 5 — שמור והצג
- אם קיימת `/mnt/user-data/outputs/` (Claude.ai/Cowork) — שמור שם `dashboard_<תיאור>.html` ואז `present_files`.
- אחרת (Claude Code מקומי) — שמור ליד קובץ הקלט או בתיקיית הורדות, והחזר את הנתיב המלא כקישור.

## בקרת איכות לפני מסירה
- כל מספר בדשבורד מופיע בקובץ המקור? (אין המצאות)
- הדשבורד עונה על המיקוד שהמשתמש ביקש, לא על שאלה אחרת?
- הפלטה שנבחרה הוחלה, והטקסט קריא על הרקע?
- ה-HTML נפתח עצמאית, הטאבים עובדים, RTL תקין, אין טקסט חתוך?
- יש לפחות תובנה אחת לא-טריוויאלית — משהו שאי אפשר לראות במבט חטוף על הקובץ הגולמי?

## הערות
- עברית עם encoding נכון (UTF-8) — חשוב במיוחד ל-CSV/טקסט עבריים.
- מסמך בלי נתון כמותי עדיין יכול להיות דשבורד: נושאים, ציר זמן, ציטוטים ומסקנות הם ויזואליזציה לגיטימית.
- קובץ ענק (עשרות אלפי שורות) — דגום/אגרגט חכם, אל תטמיע הכל ב-HTML.
