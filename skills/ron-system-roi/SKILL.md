# RON Assistant — מחשבון ROI

## מטרת הסקיל
להפוך מספרים גולמיים של פרוספקט למסמך הוכחה עסקית. ROI לא תיאורטי — חישוב קונקרטי במספרים שלו.

## הקלט שאתה מבקש מרון

שאל את 5 השאלות הבסיסיות:

1. **כמה לידים בחודש הלקוח מקבל היום?** (ממוצע 3 חודשים אחורה)
2. **כמה זמן ביום הוא מוציא על שיווק וניהול לידים?** (שעות)
3. **מה ערך הלקוח הממוצע אצלו?** (₪ לעסקה)
4. **מה שיעור הסגירה הנוכחי?** (% מהלידים שהופכים לעסקה — אם לא יודע, השתמש ב-10%)
5. **מה תעריף השעה שלו?** (אם לא יודע, השתמש ב-₪150)

## נוסחאות חישוב

### חיסכון בזמן
```
זמן נחסך בשבוע = (זמן שיווק יומי × 5 ימים) - 2 שעות (זמן ניהול המערכת)
זמן נחסך בחודש = זמן נחסך בשבוע × 4
ערך הזמן = זמן נחסך בחודש × תעריף שעה
```

### ערך מלידים נוספים
```
לידים חדשים בחודש = 30-50 (תלוי בנישה ובערים)
מהם פותחים שיחה = 30% (10-15 שיחות)
מהם הופכים לפגישות = 20% (2-3 פגישות)
מהם סוגרים = 30% (פגישה אחת חדשה לפחות)
ערך כספי = ערך לקוח ממוצע × 1 = ₪X חדשים בחודש
```

### עלות המערכת
```
Full Suite = ₪1,800/חודש
עלויות API = ₪150/חודש (ממוצע)
סך עלות = ₪1,950/חודש
```

### ROI
```
ערך כולל בחודש = ערך הזמן + ערך לקוחות חדשים
ROI = (ערך כולל - עלות) / עלות × 100
```

## דוגמת חישוב מלא

**נתוני הפרוספקט:**
- 8 לידים בחודש היום
- 2.5 שעות ביום על שיווק
- ₪3,000 ערך עסקה ממוצע
- 12% שיעור סגירה
- ₪200 תעריף שעה

**חישוב:**

```
חיסכון בזמן:
- 2.5 ש' × 5 ימים = 12.5 ש'/שבוע
- מינוס 2 ש' ניהול המערכת = 10.5 ש' נטו
- 10.5 × 4 שבועות = 42 ש'/חודש
- 42 × ₪200 = ₪8,400 ערך הזמן

ערך מלידים חדשים:
- 40 לידים חדשים בחודש מהמערכת
- 12 פותחים שיחה (30%)
- 2.4 פגישות (20%)
- 0.7 עסקאות (30%) — נעגל ל-1
- 1 × ₪3,000 = ₪3,000

סה"כ ערך:
₪8,400 + ₪3,000 = ₪11,400/חודש

עלות:
₪1,800 (מערכת) + ₪150 (API) = ₪1,950/חודש

ROI:
(₪11,400 - ₪1,950) / ₪1,950 = 485%
החזר השקעה תוך 6 ימים

שנתי:
ערך: ₪11,400 × 12 = ₪136,800
עלות: ₪1,950 × 12 = ₪23,400
רווח נטו: ₪113,400 בשנה הראשונה
```

## פלט שלך לרון

החזר:

1. **טבלת ROI מסודרת** עם כל המספרים
2. **משפט סיכום חזק** — "₪X חוסך בחודש, החזר השקעה תוך Y ימים"
3. **גרף השוואה** (טקסטואלי) — היום מול עם המערכת
4. **תרחיש שמרני + תרחיש אופטימי** — שני חישובים, שיהיה רנג'
5. **3 הסתייגויות** — מה צריך לקרות כדי שזה יעבוד (לקוח עונה ל-WhatsApp, מבצע 2 ש' שבועיות, נישה רלוונטית)

## כללי ברזל

### חובה
- להשתמש רק במספרים שהפרוספקט נתן
- להציג תרחיש שמרני (לא רק "best case")
- לכלול את עלות המערכת בגלוי
- לציין את הזמן עד החזר השקעה

### אסור
- להגזים — "ROI של 2000%" נשמע לא אמין
- להבטיח מספרים מדויקים — "תסגור 5 עסקאות בחודש"
- להתעלם מעלויות נסתרות (זמן הקדמה, עקומת למידה)
- להציג מספרים בלי הנחות יסוד שקופות

## הסתייגויות שצריך לכלול בפלט

```
⚠️ ROI זה מבוסס על הנחות:

1. הלקוח מקדיש 2 שעות שבועיות לעבודה במערכת (לחיצה על "שלח", מענה)
2. הנישה והערים שלו רלוונטיות לסריקה (תאומת בשבועיים הראשונים)
3. הוא עונה ל-WhatsApp תוך 24 שעות מקבלת תגובה
4. שיעור הסגירה הנוכחי יישמר על לידים חדשים

תקופת התאמה: 30 יום ראשונים — הנתונים יהיו נמוכים יותר
חודש 2-3: עליה ל-80% מהפוטנציאל
חודש 4+: יציבות בנתונים שמוצגים למעלה
```

```
- מפסיד ~1 עסקה בחודש שהיתה יכולה להגיע (= ₪36,000 בשנה)

```
