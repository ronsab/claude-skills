# RON DIGITAL STUDIO — מסמך פירוט שירותים

## מה הסקיל עושה

בונה קובץ HTML מקצועי (עיצוב זהב/קרם, RTL, פונט Heebo) עם 13 סעיפים קבועים.
הלקוח מקבל אותו לפני הצעת המחיר — כדי להבין בדיוק מה ייעשה ומה לא.

---

## שלב 1 — שאלון לאיסוף מידע

**לשאול את רון לפני שמתחילים. לא להמציא שום פרט.**

1. **שם הלקוח / העסק** — מה שם העסק? (לדוגמה: "קפה בלה", "ד"ר משה לוי")
2. **סוג הפרויקט** — (דף נחיתה / אתר תדמית / חנות / אוטומציה / אחר)
3. **מטרת הפרויקט** — מה הלקוח רוצה להשיג? (1-3 משפטים)
4. **עמודי האתר** — אילו עמודים יש? (ראשי, אודות, שירותים, גלריה, צור קשר...)
5. **שירותים מיוחדים** — יש פיצ'רים מיוחדים? (טופס, גלריה, סליקה, שפות...)
6. **הרחבה עתידית** — האם יש תוכנית לשלב ב׳? (חנות, בלוג, אזור לקוחות...)
7. **שם איש קשר מצד הלקוח** — לשורת החתימה (שם + תפקיד)
8. **לוח זמנים** — האם יש דד-ליין ספציפי? (אם לא — להשתמש בברירת מחדל לפי סוג)

אם חסר מידע קריטי — לשאול. **לא לנחש.**

## שלב 2 — מילוי התבנית

קרא את `template.html` בתיקיית הסקיל.
החלף כל `{{PLACEHOLDER}}` בתוכן האמיתי לפי הטבלה:

| Placeholder | מה למלא |
|---|---|
| `{{CLIENT_NAME}}` | שם הלקוח / העסק |
| `{{PROJECT_TYPE}}` | סוג הפרויקט (לדוגמה: "אתר תדמית") |
| `{{DOC_NUMBER}}` | SOW-YYYY-MM-NN (לדוגמה: SOW-2026-05-01) |
| `{{DOC_DATE}}` | תאריך היום בפורמט DD/MM/YYYY |
| `{{CLIENT_CONTACT}}` | שם + תפקיד מצד הלקוח |
| `{{PROJECT_GOAL}}` | 2-3 משפטים על מטרת הפרויקט |
| `{{SCOPE_ITEMS}}` | רשימת `<li>` לפי מה שסוכם |
| `{{PAGES_LIST}}` | בלוקי page-block — אחד לכל עמוד (ראה מבנה למטה) |
| `{{NAV_ITEMS}}` | `<li>` לכל פריט ניווט |
| `{{FOOTER_ITEMS}}` | `<li>` לכל פריט תחתית |
| `{{MOBILE_ITEMS}}` | `<li>` לכל התאמת מובייל |
| `{{FUTURE_ITEMS}}` | `<li>` לכל פריט תשתית עתידית |
| `{{STAGES_LIST}}` | בלוקי stage — אחד לכל שלב עבודה (ראה מבנה למטה) |
| `{{INCLUDED_LIST}}` | `<tr>` לכל פריט כלול |
| `{{EXCLUDED_LIST}}` | `<tr>` לכל פריט לא כלול |
| `{{FUTURE_PHASE}}` | `<li>` לכל פריט שלב ב׳ |
| `{{TIMELINE_TOTAL}}` | מספר ימי עסקים כולל (לדוגמה: "7-10") |
| `{{TIMELINE_ROWS}}` | `<tr>` לכל שלב בלוח הזמנים |
| `{{SUMMARY_TEXT}}` | 2-3 משפטי סיכום |

## מבנה page-block (סעיף 03)

```html
<div class="page-block">
  <div class="page-block-header">
    <span class="page-block-num">P.01</span>
    <span class="page-block-title">שם העמוד</span>
  </div>
  <div class="page-block-body">
    <ul class="page-block-list">
      <li>פריט תוכן 1</li>
      <li>פריט תוכן 2</li>
    </ul>
    <div class="page-block-goal"><strong>מטרה:</strong> תיאור המטרה של העמוד</div>
  </div>
</div>
```

## מבנה stage (סעיף 08)

```html
<div class="stage">
  <div class="stage-num-wrap">
    <span class="stage-num-label">שלב</span>
    <span class="stage-num">1</span>
  </div>
  <div class="stage-content">
    <div class="stage-title">שם השלב</div>
    <div class="stage-text">תיאור מה קורה בשלב זה.</div>
    <ul class="stage-list">
      <li>פריט א</li>
    </ul>
    <div class="stage-output">↳ תוצר: תיאור התוצר</div>
  </div>
</div>
```

## מבנה שורת טבלה (סעיפים 09 + 10)

```html
<!-- כלול -->
<tr>
  <td class="marker">✓</td>
  <td>תיאור הפריט הכלול</td>
</tr>

<!-- לא כלול -->
<tr>
  <td class="marker">✗</td>
  <td>תיאור הפריט שלא כלול</td>
</tr>
```

## סעיפים 9 + 10 — ברירות מחדל קבועות

**כלול תמיד (אלא אם סוכם אחרת):**
- עיצוב ממותג מותאם לעסק
- פיתוח ועלייה לאוויר
- התאמה מלאה למובייל
- SEO טכני בסיסי (meta, titles, alt)
- חיבור דומיין ואחסון
- 2 סיבובי תיקונים לאחר אישור ראשוני

**לא כלול תמיד:**
- צילום מקצועי (חומרים מסופקים ע"י הלקוח)
- כתיבת תוכן / קופירייטינג
- ניהול שוטף לאחר ההשקה
- קמפיינים פרסומיים ממומנים
- עלויות חיצוניות (דומיין, אחסון, סליקה, רישיונות תוכנה)

|---|---|
| דף נחיתה פשוט | 3-5 ימי עסקים |

| אוטומציה פשוטה | 3-5 ימי עסקים |
| אוטומציה מורכבת | 7-14 ימי עסקים |
