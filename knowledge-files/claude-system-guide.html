שאלות נפוצות

למה לפתוח Claude מתוך תיקיית הלקוח?
כשפותחים Claude Code מתוך clients/cohen-shikutzim/, הוא קורא את CLAUDE.md שם ויודע מיד: מי דני, מה הסגנון, מה ה-CTA, מה הוסכם. אין צורך להסביר בכל פעם מחדש. כמו לפתוח תיקיית לקוח אצל רואה חשבון.
מה אם לקוח לא רוצה Vercel?
/deploy עובד עם Vercel כברירת מחדל. אפשר לבקש "תן לי את הקובץ בלי דפלוי" — Claude יכין HTML להורדה ישירה.
איך Claude יודע לכתוב בסגנון שלי?
יש קובץ ron-social-dna.md שמגדיר את הקול שלך: איזה מילים מותרות ("וואלה", "בנאדם", "מטורף"), איזה אסורות ("סקייל", "מינוף"), ומה המבנה הנכון לפוסט. הקובץ הזה נטען ל-social-post-generator בכל פעם.
מה זה Context7 שמוזכר ב-CLAUDE.md?
ספריות קוד (Three.js, GSAP, Tailwind) מתעדכנות כל הזמן. Claude לפעמים מכיר גרסה ישנה. Context7 מביא לו את התיעוד העדכני בזמן כתיבת הקוד — כמו לתת למנגן שיט מעודכן לפני הופעה.
Auto-Git — האם זה לא מסוכן?
לא. ה-hook רץ רק בתיקיות שכבר מוגדרות כ-git repo. אם אין git — הוא מדלג בשקט. הוא לא יוצר repos חדשים, רק מוסיף commits לקיים. ואפשר תמיד לבטל commit עם git reset HEAD~1.
