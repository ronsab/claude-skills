# PRD — מערכת ניהול מרלוג

## 1. מטרת המערכת
לרכז את כל תהליך העבודה העסקי של מרכז לוגיסטי (מרלוג) במקום אחד: ניהול לקוחות, סוכני שטח, קטלוג מוצרים, מחירונים, הנחות, הזמנות, מלאי, מחסן, הפצה, סטטוסים, הרשאות, מסמכים, דוחות ניהוליים, אוטומציות והתראות.

**בעיות שהמערכת פותרת:** עבודה ידנית, טעויות, כפילויות, חוסר סנכרון בין מכירות למחסן, וחוסר שליטה על מלאי והזמנות.

## 2. הקשר ואילוצים
- שוק ישראלי: מטבע ש"ח, מע"מ 18%, מסמכים בעברית (תעודת משלוח / חשבונית מס).
- ממשק בעברית מלא, RTL, מותאם גם למובייל (PWA).
- קטלוג: 300 פריטים כיום, עד 4000 בהמשך.
- תחזוקה מינימלית לטווח ארוך (פלטפורמה מנוהלת).
- חשבוניות מס מונפקות דרך אינטגרציה לספק חיצוני מורשה (לא במערכת עצמה).

## 3. חוקים עסקיים מרכזיים
- מחירים שונים לפי **סוג לקוח** (פרטי/עסקי) או לפי **סוכן** (מחירונים).
- סוכן יכול לתת הנחה בעצמו **עד תקרה**; חריגה דורשת **אישור** (תפעול/בעלים).
- ההזמנה **נשלחת אוטומטית למחסן** עם האישור/השליחה.
- **מלאי מתעדכן בזמן אמת**, עם מניעת מכירת-יתר.
- קטלוג מוצג גם כ**גלריית קנייה** לפי פריט/מוצר.

## 4. משתמשים (9 תפקידים) ויכולות עיקריות
1. **בעל העסק / מנכ"ל** — דאשבורד ניהולי: מכירות לפי תקופה, הזמנות פתוחות, מלאי קריטי, ביצועי סוכנים, לקוחות פעילים/בהמתנה, הכנסות צפויות, חריגות הנחה, הזמנות תקועות, דוחות מתקדמים.
2. **מנהל תפעול** — מעקב אחר כל ההזמנות, אישור חריגות, שיוך למחסן, מעקב אספקות, פתיחת משימות מחסן/נהג, טיפול בבעיות מלאי, ניהול סטטוסים, דוחות תפעוליים.
3. **סוכן שטח** (מובייל/טאבלט/מחשב) — לקוחות שלו, פתיחת לקוח חדש, קטלוג ומחירים לפי הרשאה, ביצוע הזמנה, בקשת הנחה חריגה, סטטוס הזמנות, הערות+תמונות/מסמכים, שליחה לאישור, היסטוריית לקוח.
4. **מחסנאי** — רשימת הזמנות להכנה, סטטוס ליקוט, סימון נאסף, חוסרים, החלפת מוצר, דיווח מוצר חסר, עדכון כמות בפועל, תעודת ליקוט, סגירת טיפול.
5. **מלגזן** — משימות איסוף, מיקום מוצר, איתור מדף/אזור, אישור ביצוע, דיווח בעיה, סטטוס משימה.
6. **נהג / מוביל** — משלוחים יומיים, כתובות+אנשי קשר, ניווט, יציאה לדרך, נמסר, צילום מסירה, חתימת לקוח, דיווח כשל, החזרת סחורה.
7. **הנהלת חשבונות** — הזמנות מאושרות, הפקת מסמכים, סנכרון/שליחה לספק חיצוני, סטטוס תשלום, ייצוא אקסל.
8. **לקוח עסקי** — פורטל: כניסה, הזמנות, הזמנה חדשה, אישור הצעת מחיר, מסמכים, מעקב אספקה, העלאת מסמכים, תקשורת מול הסוכן.
9. **לקוח פרטי** — כדומה ללקוח עסקי, ללא מאפייני אשראי/ח.פ.

## 5. תכולה לפי שלבים
### MVP (שלב 1)
- ישויות ליבה: Product, Category, InventoryItem, StockMovement, Customer, Agent, PriceList/PriceListItem, Order/OrderLine, DeliveryNote, Counter, User+roles.
- גלריית קטלוג, סוכן order-taking, שליחה אוטומטית למחסן, מסך ליקוט, הפצה+POD, קליטת מלאי ידנית.
- תמחור לפי סוג-לקוח + סוכן + הנחת-סוכן עם תקרה ואישור; חישוב מע"מ.
- שריון מלאי דרך server function + ledger (StockMovement).
- תעודת משלוח PDF (RTL). חשבוניות דרך אינטגרציה לספק.
- דאשבורד בסיסי לבעלים/תפעול.

### שלב 2
סריקת ברקוד (מצלמה), פורטל לקוחות, ניתוב משלוחים רב-תחנתי, החזרות/זיכויים, מחירון לפי כמות/לקוח-ספציפי, ריבוי-מחסנים, התראות reorder, reconciliation job.

### שלב 3
אינטגרציית הנה"ח מלאה, עמלות סוכן, אנליטיקה מתקדמת, push notifications, ואופציית offline נייטיב במידת הצורך.

## 6. שאלות פתוחות ללקוח
- ספק חשבוניות/הנה"ח: **חשבונית ירוקה (Green Invoice)** — נבחר. נדרשת אינטגרציית API להנפקת חשבונית מס/קבלה וקבלת מספר הקצאה.
- כמות סוכנים מקבילים (קובע אם soft-reservation מספיק)?
- מיזוג מלגזן→מחסנאי ל-MVP?
- מחיר נטו או אחוז הנחה במחירון? מדרגות כמות / מחיר ספציפי ב-MVP?
- אכיפת מסגרת אשראי לעסקיים? COD לפרטיים?
- מקור נתוני הקטלוג (CSV/פיד) ותחזוקת תמונות?
- מחסן יחיד או מרובה כבר עכשיו?
- offline בשטח/מחסן — חסם אמיתי?
