# מערכת ניהול מרלוג — מסמכי אפיון

מסמכי האפיון לבניית מערכת ניהול מרכז לוגיסטי (מרלוג) לעסק הפצה בישראל.
המערכת נבנית על פלטפורמת **Base44** (מנוהלת, low-ops), עם נייד כ-PWA.

## תוכן
- [`PRD.md`](./PRD.md) — מטרות, משתמשים, תכולה, שלבים.
- [`data-model.md`](./data-model.md) — מודל הנתונים (entities), שדות, וקשרים.
- [`permissions.md`](./permissions.md) — מטריצת הרשאות ל-9 התפקידים.
- [`workflows.md`](./workflows.md) — מכונת-מצבים של הזמנה, לוגיקת תמחור ולוגיקת מלאי בזמן אמת.

## עקרונות ארכיטקטורה מנחים
1. **פלטפורמה מנוהלת (Base44)** — אפס DevOps, מתאים לתחזוקה מינימלית לטווח ארוך.
2. **נייד = PWA** — אותה אפליקציה, ללא pipeline נייטיב נפרד.
3. **לוגיקה רגישה ב-server functions בלבד** — תמחור, שריון מלאי, מעברי-סטטוס ומספור מסמכים אינם רצים בצד-לקוח. זה מרכז את הקושי במספר מצומצם של פונקציות ועוקף את היעדר הטרנזקציות.
4. **Snapshot של מחיר/מע"מ בשורת ההזמנה** — מסמכים היסטוריים נשארים immutable גם כשהמחירונים משתנים.
5. **מסמכים — גישה משולבת** — הצעת מחיר ותעודת משלוח מופקות בתוך המערכת (PDF RTL); חשבונית מס וקבלה דרך **חשבונית ירוקה (Green Invoice)** (מספר הקצאה, תאימות רשות המסים). הצעת מחיר באישור → המרה אוטומטית להזמנה.
